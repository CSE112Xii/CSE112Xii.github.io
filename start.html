Download PeterBook

Build Setup

Step One

After downloading the ZIP file, extract all to the current folder or to a folder that you wish.

We must now set the working directory to PeterBook in our terminal:

cd ~/Downloads/xii/XiiFrontendVue/peterbook/

If the ZIP file is extracted to a different folder, make sure to set to the correct working directory by replacing 'REPLACE-FOLDER' below:

cd ~/REPLACE-FOLDER/xii/XiiFrontendVue/peterbook/

Step Two

After setting our current working directory to 'peterbook', install dependencies:

npm install

This will install dependencies into the 'node_modules/' directory of our node project
Once the installation report with 'Success', move onto step two.

If you need help getting started with using 'npm' and/or would like to get more information about 'npm', follow this link

Step Three

npm run dev

or

npm start

We are serving at localhost:8080 with hot reload.
Hot reload will keep the app running while injecting new versions of files edited at runtime.
This will keep from losing any of the state when tweaking UI.

Step Four

We will now prep tasks for our project specified from our 'package.json' file:

npm run build

We have build for production with minification; however, if you would like to build for production and view the bundle analyzer report:

npm run build --report

Step Five

We will now run all of our tests:

npm test

To run certain tests instead:

npm run unit to run unit tests

npm run e2e to run end-to-end tests

End-to-end tests will visually see our interaction of our build between two users in real-time

Why Vue?

We started with React

In the beginning, our front-end team was comfortable with React. However, we decided that switching to Vue would allow more people to help on the front-end code because this project seemed very front-end focused and Vue is easier to learn.

Outcome

Although this had more members productive, the front-end would have been finished within a few days by one of our front-end team (Vincent) since he was very experienced. In the end, we made a collaborative effort to create a working project.

Pros & Cons of Vue

Pros of Vue: Lightweight, “easy to learn” - this would allow for future classes to pick up our codebase faster.
Cons of Vue: Takes time to learn (especially best practices). Also takes time to set up (had to take time to set up a new codebase rather than working off the React codebase already set up).

Conclusion

Basically, switching to Vue would have been a better idea if we done earlier in the quarter; especially when started with another framework. If switched earlier, most of the cons listed above wouldn’t really matter, and would get the huge gain of a lightweight, easy to learn codebase.
